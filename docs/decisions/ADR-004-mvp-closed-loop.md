# ADR-004：首版先交付最小可信闭环

## 状态

Accepted

## 日期

2026-06-25

## 背景

项目完整愿景包含采集、检索、AI 工作流、图表、报告编辑、导出和邮件交付。如果首版同时实现 RSS/API/网页采集、Playwright 登录态采集、Elasticsearch 混合检索、图表编辑、PDF 和 SMTP，交付风险会集中在多个不稳定边界上，难以及早验证核心价值。

核心价值是把用户资料和规则转化为可追溯、可审核、可导出的正式报告。因此首版应先证明这条闭环可信。

## 决策

首版优先实现：

- 登录和角色权限。
- 创建 `ResearchBrief`。
- 上传或粘贴用户资料。
- 生成带引用的报告草稿。
- 人工编辑、审核和单章节重生成。
- Markdown 和 DOCX 导出。

以下能力延后到 MVP 闭环稳定后：

- RSS/API/网页和 Playwright 登录态采集。
- Elasticsearch 完整混合检索和聚类。
- 图表编辑和 PDF 导出。
- SMTP 正式发送和失败重试。
- 历史报告模板自动抽取。

## 后果

- MVP 可以更快验证“可追溯报告生产”是否成立。
- 数据模型、API 和工作流仍保留扩展点，避免后续重写核心边界。
- 部分导航和文档能力会先以占位或增强阶段形式存在。
- 验收标准必须区分 MVP 验收和增强功能验收，避免范围漂移。
