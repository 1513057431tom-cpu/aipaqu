# ADR-002：数据存储职责分离

## 状态

Accepted

## 日期

2026-06-25

## 决策

- MySQL 是业务事实来源。
- Elasticsearch 负责文档、分块和检索，不保存唯一业务状态。
- Redis 负责短期缓存、队列、锁、进度和幂等。
- 文件存储负责原文件、网页快照和导出物。

## 理由

报告版本、审核和交付需要关系事务；全文与向量检索更适合 Elasticsearch；队列和瞬时状态适合 Redis；大型文件不应放入关系数据库。

## 后果

- Elasticsearch 和 Redis 故障后可从 MySQL 与文件存储恢复。
- 删除与备份策略必须跨存储协调。
- 文档索引需要明确版本和重建任务。
