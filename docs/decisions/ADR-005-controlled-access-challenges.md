# ADR-005：访问挑战采用人工接管与授权记录

## 状态

Accepted

## 日期

2026-06-26

## 背景

资料采集软件会遇到验证码、付费墙、登录权限、MFA、403/429 和 robots 限制。直接实现自动破解或绕过会带来法律、合规、账号安全和平台封禁风险，也会破坏项目“可追溯、可审计、可复现”的产品定位。

同时，产品确实需要支持用户有权访问的登录或付费来源，例如用户自有账号、机构订阅、内部系统和授权数据库。

## 决策

系统支持受控访问挑战处理：

- `AccessGrant` 记录来源授权、用途、有效期和确认人。
- `AccessChallenge` 记录验证码、付费墙、登录失效、MFA、403、429、robots 等挑战。
- `ManualVerificationSession` 允许人工在受控浏览器中完成合法登录、订阅确认或验证码输入。
- 任务遇到访问挑战后暂停，等待人工处理或进入可解释失败状态。
- 系统不保存验证码答案、MFA 一次性密码、密码明文或绕过脚本。

系统不实现：

- 自动破解验证码。
- 绕过付费墙、登录授权、IP 封禁或风控。
- 自动规避 robots 或站点条款。
- 执行用户提交的任意脚本来对抗访问控制。

## 后果

- 能服务合法登录和付费来源采集场景。
- 采集失败原因可解释、可审计、可恢复。
- 避免将产品能力定位为访问控制绕过工具。
- 后续实现必须把挑战识别、人工接管、授权记录和审计作为采集增强的一部分。
