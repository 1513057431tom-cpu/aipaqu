# ADR-001：首版采用模块化单体与独立 Worker

## 状态

Accepted

## 日期

2026-06-25

## 背景

首版面向个人和小团队，但同时包含采集、AI 工作流、检索、报告和交付。全部拆为微服务会增加部署、调试和接口治理成本；全部放在单进程又无法隔离长任务和浏览器任务。

## 决策

采用模块化单体：

- 一个 FastAPI API 应用。
- 一个共享领域和数据访问代码库。
- Celery 普通 Worker 与 Playwright 爬虫 Worker 分进程运行。
- 通过模块接口隔离采集、模型、工作流、存储和交付。

## 后果

- 单机 Docker Compose 易于部署和排障。
- 长任务不会阻塞 API。
- 未来可按队列和模块逐步拆服务。
- 必须保持模块边界，禁止跨模块直接修改内部表和状态。
