# 产品需求：智能情报与周期报告平台

## 1. 产品目标

为个人和小团队提供一个可配置的情报生产平台，将“提出研究需求、采集资料、补充内部数据、AI 联合分析、生成图表和报告、人工审核、导出发送”整合成可重复运行的工作流。

核心价值不是简单总结网页，而是根据用户的文档规则，将多来源证据转化为结构稳定、数据可信、引用可追溯的正式报告。

## 2. 目标用户

- `ADMIN`：配置系统、用户、模型、采集凭据、SMTP 和工作流。
- `EDITOR`：编写采集需求、上传资料、运行任务、编辑和审核报告。

首版面向单个工作空间，不实现商业化多租户、套餐和计费。

## 3. 首版实施策略

首版采用“最小可信闭环”优先，而不是一次性实现所有采集、图表和交付能力。MVP 必须先跑通：

```text
登录
  → 创建 ResearchBrief
  → 上传或粘贴用户资料
  → 生成带引用的报告草稿
  → 人工编辑与审核
  → Markdown / DOCX 导出
```

增强功能在 MVP 闭环稳定后逐步加入，包括 RSS/API/网页采集、Elasticsearch 混合检索、ECharts 图表编辑、SMTP 正式发送、Playwright 登录态采集和复杂模板抽取。

## 4. 核心使用场景

### 3.1 创建采集与分析需求

用户在“采集需求与数据工作台”创建 `ResearchBrief`，填写：

- 研究主题、背景、目标和报告用途。
- 必须采集的内容、关键词、同义词和排除词。
- 指定网站、栏目、URL、RSS 或 API。
- 数据时间范围和报告周期。
- 重点指标、分析维度、数据口径和单位。
- 必须回答的问题、禁止推断的内容和输出要求。
- 资料优先级：核心依据、补充参考、背景资料、排除内容。

需求可以保存为模板并复制复用。

### 3.2 选择时间范围

支持以下时间模式：

- 指定单日：选择一个年月日。
- 自然周：默认周一 00:00 至周日 23:59:59。
- 自然月：当月第一天至最后一天。
- 自定义期间：包含开始日和结束日。
- 定时日报、周报和月报：运行时自动计算所属周期。

默认时区为 `Asia/Shanghai`。系统分别保存：

- `collectionWindow`：允许采集或查询的数据时间范围。
- `analysisWindow`：报告统计和分析采用的时间范围。

资料可以被采集但因不属于分析期间而不进入本期结论。

### 3.3 添加用户资料与数据

用户可以：

- 粘贴文本、URL 和表格数据。
- 上传 URL/网站清单。
- 上传 PDF、DOCX、TXT、Markdown。
- 上传 XLSX、CSV、JSON。
- 为上传项填写来源、日期、数据口径、单位、可信度和用途。

上传资料与网络采集资料统一进入预处理、检索和分析流程，但必须保留来源类型。

### 3.4 配置报告规则

用户可上传历史报告或模板，并补充文字要求。系统抽取并生成可编辑的规则版本：

- 报告类型和适用场景。
- 标题、章节顺序和必填章节。
- 语气、语言、篇幅和读者。
- 必须展示的指标、表格和图表。
- 引用格式、来源数量和事实核验要求。
- DOCX/PDF 页面和样式规则。

已生成报告永久绑定当时使用的规则版本。

### 3.5 生成、审核和交付

- 系统生成报告草稿、引用、数据集和图表。
- 用户可以编辑正文、修改图表、查看来源、比较版本。
- 支持重新生成单个章节，不覆盖人工修改的其他章节。
- 审核通过后可导出 DOCX、PDF、Markdown。
- 支持通过 SMTP 发送正文和附件，记录发送结果并可重试。

## 5. 功能范围

### MVP 必须实现

- 账号登录和 `ADMIN`/`EDITOR` 权限。
- 采集需求与数据工作台的最小闭环：基本信息、时间范围、必答问题、上传资料、模板选择、运行前校验。
- 日、周、月、自定义期间。
- 文档、文本、URL 清单和结构化数据上传；MVP 可先将 URL 清单作为用户资料保存，不强制联网采集。
- 用户资料解析、清洗、时间过滤、去重和可检索文本化。
- 手工创建或编辑报告规则版本；范例文档自动抽取可延后。
- 平台报告工作流的核心节点：Brief、Preprocess、Research、Analyze、Summarize、Review、Compose、Human Review、Export；具体执行器不得成为业务 API 或数据库公共契约。
- 报告编辑、引用溯源、人工审核和单章节重生成。
- Markdown 和 DOCX 导出。

### MVP 后增强

- RSS、开放 API、静态网页和授权登录网页采集。
- Elasticsearch BM25、向量和结构化过滤的完整混合检索。
- 报告模板从历史 DOCX/PDF 自动抽取。
- ECharts 图表编辑、图表图片渲染和数据口径可视化。
- PDF 导出。
- SMTP 测试发送、正式发送、日志和失败重试。
- Playwright 登录态采集、凭据异常修复和复杂来源调度。

### 首版不实现

- 商业化多租户、套餐、支付和用量计费。
- 任意代码式爬虫市场。
- 复杂部门审批链。
- 企业微信、钉钉和飞书实际发送。
- Kubernetes 和跨区域高可用。

## 6. 质量与验收标准

### MVP 验收

- 相同 URL 或相同正文不会重复入库。
- 自定义期间包含起止日期，周和月边界计算正确。
- 用户上传文档、粘贴文本和结构化数据能够共同参与报告分析。
- 每条关键结论至少关联一个可查看的证据；无证据内容必须明确标为推断或待核实。
- 导出 Markdown 和 DOCX 后，标题、章节、引用和人工修改内容保持一致。
- 报告生成失败可从失败节点继续，不必重新采集全部数据。
- 幂等重试不会重复生成报告版本。
- 新增采集器、模型供应商、Agent 节点、工作流执行器或交付通道不需要修改核心业务流程。

### 增强功能验收

- 网络资料、用户上传文档和结构化数据能够共同参与报告分析。
- 图表保存原始数据集、单位、口径和来源。
- PDF 导出与 DOCX 的章节、引用和图表内容一致。
- SMTP 幂等重试不会重复发送邮件。
- 登录态采集凭据失效时会暂停对应来源并给出可操作修复建议。
