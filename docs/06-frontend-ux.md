# 前端信息架构与交互

## 1. 主导航

1. `Dashboard`：任务、报告、异常和用量概览。
2. `采集工作台`：编写需求、时间范围、来源、资料和数据。
3. `情报中心`：文档检索、聚类、来源和证据查看。
4. `报告中心`：草稿、审核、发布、导出和发送。
5. `深度分析`：数据集、指标、图表和专题分析。
6. `实时监控`：采集任务、工作流节点、错误和重试。
7. `模板中心`：报告规则和版本。
8. `系统配置`：模型、Embedding、SMTP、用户和安全。

## 2. MVP 页面清单

MVP 先实现以下页面，其他导航可以显示为禁用入口或“后续版本”占位：

| 页面 | 路由建议 | 核心目标 |
| --- | --- | --- |
| 登录 | `/login` | 建立安全 Cookie 会话 |
| Dashboard | `/` | 查看最近任务、待审核报告和异常 |
| 采集工作台 | `/briefs/new`、`/briefs/{id}` | 创建 Brief、上传资料、运行任务 |
| 任务监控 | `/tasks/{id}` | 查看节点进度、错误和重试入口 |
| 报告编辑 | `/reports/{id}` | 编辑正文、查看引用、提交审核、批准 |
| 导出记录 | `/reports/{id}/exports` | 创建并下载 Markdown/DOCX 导出 |
| 系统配置 MVP | `/settings` | 查看当前用户、模型和基础配置状态 |

所有 MVP 页面必须具备：

- 加载状态。
- 空状态。
- 错误状态和重试入口。
- 权限不足状态。
- 成功反馈。
- 表单字段级校验信息。

## 3. 采集需求与数据工作台

采用分步式编辑，但允许随时保存草稿：

```text
基本信息
  → 时间范围
  → 采集内容
  → 网站与 URL
  → 上传资料
  → 上传/录入数据
  → 分析与报告规则
  → 检查并运行
```

### 关键交互

- 时间范围提供“单日、周、月、自定义期间”切换，并实时展示解析后的起止时间。
- 采集内容支持关键词标签、排除词、必答问题和自由文本说明。
- 来源支持粘贴多行 URL、上传 URL 清单和选择已保存数据源。
- 上传后显示解析状态、来源类型、记录数、字段、单位和错误。
- 运行前展示预计采集源、时间范围、材料数、数据集数和模板版本。

### 表单校验

- 标题必填，最大 120 字。
- 目标说明必填，最大 2,000 字。
- 必答问题至少 1 条，最多 20 条。
- 自定义时间范围必须满足 `startDate <= endDate`。
- 周/月范围必须显示解析后的工作空间时区。
- 上传资料必须显示解析状态：`UPLOADED`、`PARSING`、`READY`、`FAILED`。
- 运行按钮只在 Brief 达到 `READY` 且至少存在一个资料来源时可用。

## 4. 报告编辑器

三栏布局：

- 左侧：章节目录、状态和问题数量。
- 中间：报告正文、表格和图表。
- 右侧：引用证据、审核问题、版本和重生成设置。

编辑器必须支持：

- 保存人工版本。
- 重新生成选中章节。
- 点击引用打开来源摘要和原文位置。
- 图表切换类型、字段、单位、排序和颜色。
- 比较 AI 草稿与人工修改。
- 提交审核、退回修改和批准发布。

### MVP 最小交互

- 点击章节目录滚动到对应章节。
- 点击引用在右侧打开来源摘要、引用片段和原文定位信息。
- 人工编辑后保存为新的 `ReportVersion`。
- 提交审核后报告进入 `IN_REVIEW`。
- 批准后报告进入 `APPROVED`，导出按钮可用。
- 自动重生成只能覆盖用户明确选择的章节。

## 5. 实时监控

- 以时间轴展示 13 个工作流节点。
- 显示当前节点、进度、耗时、文档数、Token 和错误。
- 支持暂停、恢复和从失败节点重试。
- 登录凭据失效、来源被限流、文件解析失败应显示可操作的修复建议。

## 6. 体验和可访问性

- 桌面端优先，最低支持 1280px；核心查看页面适配平板。
- 所有表单有明确标签、帮助文本和错误信息。
- 不仅通过颜色表达状态，同时提供文字和图标。
- 所有可交互元素支持键盘操作。
- 列表具备加载、空状态、错误和重试状态。
- 避免通用 AI 风格；以信息密度、证据可读性和工作效率为设计重点。

### 1280px 桌面验收

- 主导航、页面标题、主要操作按钮在首屏可见。
- 表格和三栏编辑器不得出现文字重叠。
- 报告编辑器三栏宽度稳定：左栏 240-280px，中栏自适应，右栏 320-380px。
- 关键按钮文字不得换行或溢出。
- 状态、错误和引用标记不仅依赖颜色，还必须有文字或图标。
- 任何弹窗都必须支持键盘关闭和焦点回到触发元素。
